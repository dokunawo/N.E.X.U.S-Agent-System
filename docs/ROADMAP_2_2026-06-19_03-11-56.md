# ROADMAP 2 - 2026-06-19 03-11-56

This is the second planning log for the R.A.M.B.O. project.

## User Decisions Captured

- The system name should be R.A.M.B.O., not plain Rambo.
- Spoken wake form can remain "Rambo."
- R.A.M.B.O. should be an operating system that houses multiple agents.
- One agent can be a personal assistant, but the overall system is larger.
- The system should help with everyday problems, business ventures, technical work, situational problems, personal organization, and financial thinking within safe limits.
- Desired personality: calm command center, sharp operator, creative partner, guardian.
- Tagline: Autonomy. Precision. Execution.

## Wake Phrase Decisions

Primary:

```text
Rambo, come online.
```

Sub phrases:

```text
Rambo, status.
Rambo, lock in.
Rambo, stand by.
```

Natural starts like "Rambo..." should be supported later.

The wake language should imply Sir is the owner and R.A.M.B.O. responds to his command.

## Agent Roster Decisions

- R.A.M.B.O.: operator and overseer of the whole system.
- Architect: strategy and planning.
- Engineer: coding agent.
- Seeker: research agent.
- Analyst: data analyst agent.
- Sentinel: safety and guardian agent.
- Link: API and integration agent.
- Keeper: storage, database, and memory agent.
- Echo: conversation agent.
- Pilot: task manager agent.

Only R.A.M.B.O. should provide final advice after combining agent outputs.

Each agent should only touch what it needs.

## Dashboard Direction

The first view should be:

- Professional.
- Techy.
- Inspired by Jarvis-style command screens.
- Dense and operational.
- Focused on command, awareness, and visual presentation.

First screen should show:

- Today's priorities.
- System status.
- Agent activity.
- Inbox items.
- Risks.
- Opportunities.
- Errors that need attention.
- Next steps.
- Completed tasks.

## Data Source Direction

Best first sources:

- Calendar.
- Gmail.
- Notes.
- Tasks.
- Reminders.
- Manual entries.
- Spoken entries later.

Data can be live, historical, manual, or verbal.

Bank information should wait until permission and privacy controls are much stronger.

## Permission Direction

R.A.M.B.O. should ask before:

- Deleting files.
- Spending money.
- Contacting people.
- Sending messages.
- Sending emails.
- Exposing information.
- Connecting high-risk accounts.
- Making broad system-wide changes.

R.A.M.B.O. can make beneficial reversible changes to its own project settings, but it should say what changed and keep changes reversible.

Approval methods should include:

- Spoken confirmation.
- One-click approval.
- Written confirmation.
