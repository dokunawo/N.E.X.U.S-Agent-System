# N.E.X.U.S Roadmap

This is the living roadmap for the N.E.X.U.S operating system.

## Phase 1: Foundation

Status: in progress.

Goal:

Create the basic local command center.

Included now:

- Browser dashboard.
- FastAPI backend.
- Mock analytics.
- Simple N.E.X.U.S orchestrator.
- Early agent activity feed.
- Suggestions.
- SQLite memory log.
- Optional OpenAI bridge.

## Phase 2: Operating Blueprint

Goal:

Turn N.E.X.U.S from a visual demo into a planned operating system.

Decisions:

- Name: N.E.X.U.S.
- Tagline: Everything Connects Here.
- Personality: calm command center, sharp operator, creative partner, guardian.
- Primary wake phrase: "Nexus, come online."
- Agent roster: Architect, Engineer, Seeker, Analyst, Sentinel, Steward, Link, Keeper, Echo, Pilot.

## Phase 3: Task, Agent, Log, And Approval Structure

Goal:

Create the common language of the system.

Needed structures:

- Agent profile.
- Task record.
- Agent activity log.
- Memory entry.
- Approval request.
- Risk level.
- Data source record.

Why this matters:

The dashboard, backend, agents, memory, and voice layer all need to agree on what a task is and what an agent action means.

## Phase 4: First Real Data Sources

Goal:

Connect useful personal context.

Best early sources:

- Calendar.
- Gmail.
- Notes.
- Tasks or reminders.
- Manual entries by text or voice.

Wait on:

- Bank accounts.
- Spending actions.
- Sending messages automatically.
- Anything that exposes private information externally.

## Phase 5: Smarter N.E.X.U.S Reasoning

Goal:

Make N.E.X.U.S combine information from the agents and give useful advice.

Examples:

- "Here are today's priorities."
- "Here are the risks that need attention."
- "Here are opportunities you may be missing."
- "Here is what changed since yesterday."

## Phase 6: Improved Dashboard

Goal:

Make the first screen dense, operational, and visually sharp.

First-view sections:

- Today's priorities.
- System status.
- Agent activity.
- Inbox items.
- Risks.
- Opportunities.
- Errors needing attention.
- Next steps.

Visual direction:

- Dark command center.
- Clean panels.
- Subtle cyan accents.
- Readable text.
- Professional tech feel.

## Phase 7: Voice Layer

Goal:

Let you speak to N.E.X.U.S and hear it respond.

Wake phrase set:

- "Nexus, come online."
- "Nexus, status."
- "Nexus, lock in."
- "Nexus, stand by."

Voice should come after the dashboard and data loop are useful.

## Phase 8: Self-Learning And Self-Improvement

Goal:

Allow N.E.X.U.S to improve over time while staying controlled.

Examples:

- Learn your preferences.
- Notice repeated tasks.
- Suggest better workflows.
- Improve dashboards.
- Recommend automation ideas.

Safety rule:

Self-improvement should be logged, reversible, and approval-gated when it changes important behavior.

## Phase 9: Steward Financial Tracking

Goal:

Give N.E.X.U.S a dedicated financial tracking and budgeting agent.

Steward should help with:

- Budgets.
- Spending awareness.
- Savings goals.
- Investment awareness.
- Manual expense tracking.
- Future Notion expense tracking after approval.

Safety rule:

Steward can track, analyze, and guide. It cannot move money, trade, connect financial accounts, or make financial decisions without explicit approval.

## Phase 10: Hybrid Cloud And Local Execution

Goal:

Eventually run N.E.X.U.S as a serious operating system.

Cloud can handle:

- Always-on orchestration.
- Scheduled monitoring.
- API integrations.
- Memory sync.

Local can handle:

- Private files.
- Local scripts.
- Desktop automation.
- Sensitive data.
