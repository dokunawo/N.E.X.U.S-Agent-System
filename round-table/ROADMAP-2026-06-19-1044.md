## 1. The Round Table — Council UI

**What:** A 14-page static site replacing the single mock dashboard with
one dedicated page per agent, plus two system-level views.

- `index.html` — Round Table hub. Eleven seats arranged radially around a
  rotating 3D core (Three.js wireframe icosahedron + dual rings + particle
  field). Each seat is a clickable node that opens that agent's full page.
- 11 agent pages (`agents/*.html`) — N.E.X.U.S, Architect, Engineer,
  Seeker, Analyst, Sentinel, Steward, Link, Keeper, Echo, Pilot. Each is a
  genuinely separate page load (not a SPA route) with: role description,
  permission tier badge, live-animated stat counters, core directives,
  and a three-column Mission Journal (Completed / In Progress / Queued).
- 2 system pages (`system/*.html`) — Approvals (Sentinel's cross-agent
  queue) and Learning (N.E.X.U.S's own retrospective log).
- Visual direction: dark HUD command-center aesthetic — Signal Cyan
  glow on void black, role-specific accent colors per agent, Rajdhani
  display type + JetBrains Mono for data/timestamps + Inter for body
  copy. Built to the "futuristic holographic command center" genre
  generally — not a reproduction of any copyrighted character or brand.
- All journal/description content lives in one file, `data/agents.json`.
  `build_pages.py` regenerates all 14 pages from it — edit data, not HTML.

**Files added:** `index.html`, `agents/*.html` (11), `system/*.html` (2),
`assets/style.css`, `assets/app.js`, `assets/core3d.js`, `data/agents.json`,
`build_pages.py`, `README.md`.

## 2. Steward Budget Planner

**What:** Full manual-entry budgeting module, embedded on the Steward
agent page (`agents/steward.html`), built directly against the category
structure in your "2026 Annual Personal Financial Planner" Google Sheet
so the categories match exactly, not approximated.

- **Income** — editable rows (Paycheck, Interest, Dividends + add custom).
- **Expenses** — all 30 categories from your sheet (Asset → Gifts),
  Budgeted vs. Actual columns, live Remaining and % of Income per row.
- **Savings Goals** — target/current/target-date cards with progress bar
  and monthly-amount-needed calculation.
- **Investments** — balance, monthly contribution, assumed return →
  projected value at 1/5/10 years (standard compound-interest math).
- **Debt Payoff** — balance, APR, minimum payment → months to payoff
  (amortization formula), flags payments that don't even cover interest.
- **5 live KPI cards** — Total Income, Total Expenses, Net Cash Flow,
  Savings Rate, Cash Runway (months of expenses covered by Checking +
  Savings — seeded from your sheet's actual $157 / $210 balances).
- **Formula Reference panel** — every calculation shown two ways: the
  live JS version and its exact Excel/Sheets equivalent (`=SUM(...)`,
  `=FV(...)`, `=NPER(...)`, etc.), so the same math works if you ever
  want to keep using the spreadsheet too.

**Deliberately not built yet:** Recurring Expenses auto-scheduling, an
All Transactions log, and Account/Tag Search — those map to your sheet's
remaining tabs but depend on a real transaction feed (bank or Notion),
which AGENTS.md correctly gates behind a stronger approval system. Built
the manual layer first, per that same doc's working style: "small,
understandable steps over big rewrites."

**Files added:** `assets/budget.js` + the budget section inside
`agents/steward.html`.

## 3. Build Tooling

`build_pages.py` — Python 3, no dependencies. Reads `data/agents.json`,
generates the sidebar nav, the radial node math for the hub page, and
every agent/system page from one shared template. Re-running it after
editing the JSON is the supported way to update content going forward.
